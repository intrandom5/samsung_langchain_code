# 키키테크 Q&A 챗봇 가이드라인

## 폴더 구조

```
kiki_agent/
├── main.py              ← 실행 진입점
└── agent/
    ├── tools.py         ← 검색 도구 (search_employee, search_product, search_policy)
    ├── middleware.py    ← 가드레일, 로깅
    ├── agent.py         ← sub-agent 3개 + supervisor 정의
    └── agents.md        ← supervisor 시스템 프롬프트
```

---

## 실행 방법

터미널에서 `kiki_agent/` 폴더로 이동한 뒤 실행합니다.

```bash
cd day2/05_MAS/kiki_agent
python main.py
```

---

## 코드 흐름

`python main.py`를 실행하면 아래 순서로 코드가 동작합니다.

```
main.py 실행
  └── agent.py 불러오기
        ├── tools.py 불러오기    → RAGRetriever 초기화, search tool 3개 정의
        ├── middleware.py 불러오기 → guardrail, tool_logger 정의
        ├── model 생성
        ├── sub-agent 3개 생성  (employee / product / policy)
        ├── ask_*_agent tool 3개 정의
        └── supervisor 생성
  └── 채팅 루프 시작
```

## 파일별 역할

### tools.py — 검색 도구

각 sub-agent가 실제로 사용하는 검색 기능을 정의합니다.

| tool | 방식 | 담당 |
|---|---|---|
| `search_employee` | pandas 직접 조회 | 임직원 이름으로 정보 검색 |
| `search_product` | RAG (PDF + PPT) | 제품 카탈로그, 회사 소개 검색 |
| `search_policy` | RAG (Word) | 사내 규정 검색 |

RAG는 파일을 임베딩하는 과정이 있어서 **처음 실행할 때 시간이 걸립니다.**

### middleware.py — 미들웨어

| 미들웨어 | 종류 | 역할 |
|---|---|---|
| `guardrail` | `@before_agent` | 키키테크와 무관한 질문을 agent 실행 전에 차단 |
| `tool_logger` | `@wrap_tool_call` | tool 호출 시 이름과 결과를 출력 |
| `token_tracker` | `@after_agent` | 매 응답마다 토큰 사용량을 출력 |

### agent.py — 에이전트 정의

```
model
  ├── employee_agent  (tools: search_employee)
  ├── product_agent   (tools: search_product)
  └── policy_agent    (tools: search_policy)

supervisor (tools: ask_employee_agent, ask_product_agent, ask_policy_agent)
           (middleware: guardrail → tool_logger → token_tracker)
```

supervisor는 직접 검색하지 않고, 질문을 보고 알맞은 sub-agent에게 라우팅만 합니다.

## 자주 헷갈리는 부분

### `from .tools import ...` 에서 `.`이 뭔가요?

`.`은 **지금 이 파일이 속한 폴더**를 의미합니다.

```
agent/          ← 이 폴더가 "."
├── agent.py    ← 지금 내가 있는 곳
├── tools.py    ← .tools
└── middleware.py ← .middleware
```

`from .tools import search_employee` = **"같은 폴더의 tools.py에서 search_employee를 가져와"**

### 왜 파일 경로를 `os.path.dirname(__file__)`로 쓰나요?

`open("agents.md")`처럼 쓰면 **터미널 실행 위치 기준**으로 파일을 찾습니다.
다른 폴더에서 실행하면 파일을 못 찾는 문제가 생깁니다.

```python
# 위험: 실행 위치에 따라 경로가 달라짐
open("agents.md")

# 안전: 항상 이 파일(agent.py) 기준으로 찾음
current_dir = os.path.dirname(__file__)
open(os.path.join(current_dir, "agents.md"))
```

`__file__`은 파이썬이 자동으로 넣어주는 값으로, **지금 실행 중인 이 파일의 경로**를 담고 있습니다.

## 수정 포인트

| 목적 | 수정 파일 | 수정 내용 |
|---|---|---|
| sub-agent 역할 바꾸기 | `agent.py` | `system_prompt` 수정 |
| supervisor 역할 바꾸기 | `agents.md` | 내용 수정 |
| 새로운 문서 추가하기 | `tools.py` | `*_rag.add_file(...)` 추가 |
| 가드레일 조건 바꾸기 | `middleware.py` | `guardrail` 함수 내 프롬프트 수정 |
